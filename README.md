# What the hell — Универсальный шаблон для создания гильдий

[![Laravel](https://img.shields.io/badge/Laravel-FF2D20?style=for-the-badge&logo=laravel&logoColor=white)](https://laravel.com)
[![Vue.js](https://img.shields.io/badge/Vue.js-4FC08D?style=for-the-badge&logo=vue.js&logoColor=white)](https://vuejs.org/)

Многофункциональный веб-шаблон для игровых гильдий. Изначально создавался для гильдии *"What the Hell"* в MMORPG Royal
Quest, но теперь это готовое решение для любой игры с таймерами боссов, системой прав и базой знаний.

## 🚀 Основные возможности

- ⏱ **Таймеры возрождения боссов** — автоматический подсчёт времени до респавна с визуальным отображением
- 🔔 **Система уведомлений** — оповещения о скором появлении боссов
- 📜 **История убийств** — логирование всех побед над боссами
- 🛠 **Управление таймерами** — ручная корректировка времени, добавление новых боссов
- 👥 **Ролевая модель** — разделение прав на просмотр и редактирование
- 📚 **База знаний** — раздел с гайдами, тактиками и полезной информацией для участников гильдии

## 📦 Обновление на хостинге

1. Переключитесь на ветку `master` и убедитесь, что она в актуальном состоянии
2. Локально выполните сборку фронтенда: `npm run build`
3. На сервере перейдите в папку проекта и выполните: `git pull`
4. Примените миграции БД: `php artisan migrate`
5. При необходимости запустите сидеры: `php artisan db:seed`
6. Удалите старую папку `build` и загрузите новую (из шага 2)
7. Обновите папку `assets`, если были изменения
8. Выполните оптимизацию: `php artisan optimize`

## 🖥 Локальный запуск проекта

1. Клонируйте репозиторий:  
   `git clone https://github.com/Linsaym/WhatTheHell`

2. Установите зависимости:  
   `composer install`  
   `npm install`

3. Создайте файл `.env` из `.env.example` и сгенерируйте ключ:  
   `php artisan key:generate`

4. Выполните миграции:  
   `php artisan migrate`

5. Заполните БД тестовыми данными (при необходимости):  
   `php artisan db:seed`

## ⚙️ Требования к окружению

- [Composer](https://getcomposer.org/)
- [Node.js](https://nodejs.org/)
- Веб-сервер с поддержкой PHP (локально рекомендуется использовать [XAMPP](https://www.apachefriends.org/ru/index.html))
- База данных: MySQL или SQLite (проект использует ORM, поэтому совместим с обеими СУБД)
